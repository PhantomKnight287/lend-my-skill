# Authentication

This page will give you a brief overview of how to authenticate with the API.

## Login

The login route is at `/auth/login`. It accepts a POST request with the following parameters:

- `email` - The email address of the user
- `password` - The password of the user

Response

```json
{
  "user": {
    "role": "Freelancer",
    "username": "phantomknight287",
    "id": "clf40vl0t0000ewroucp805xe",
    "name": "Gurpal Singh",
    "profileCompleted": true
  },
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6ImNsZjQwdmwwdDAwMDBld3JvdWNwODA1eGUiLCJyb2xlIjoiRnJlZWxhbmNlciIsImlhdCI6MTY3ODk0NTM5Mn0.SpE-8ArMpDOHic6EgqpSUPkxM5h6vNPLxEr8CNvN7rQ"
}
```

The token will be used for further requests to the API.

## Register

The register route is at `/auth/register`. It accepts a POST request with the following parameters:

- `email` - The email address of the user
- `password` - The password of the user
- `confirmPassword` - The password of the user
- `country` - The country of the user(optional, Default set to India)
- `role` - The role of the user(either `Freelancer` or `Client`)
- `name` - The name of the user

Response:

```json
{
  "user": {
    "role": "Freelancer",
    "username": "phantomknight287",
    "id": "clf40vl0t0000ewroucp805xe",
    "name": "Gurpal Singh",
    "profileCompleted": false
  },
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6ImNsZjQwdmwwdDAwMDBld3JvdWNwODA1eGUiLCJyb2xlIjoiRnJlZWxhbmNlciIsImlhdCI6MTY3ODk0NTM5Mn0.SpE-8ArMpDOHic6EgqpSUPkxM5h6vNPLxEr8CNvN7rQ"
}
```
The token will be used for further requests to the API.
