# Upload

This route will be used to upload content like images,videos, documents etc to the server.

## Uploading Single File

Make a `multipart/form-data` request to the `/upload` route with the file to be uploaded in the `file` field.

<Callout type="warning">
  <strong>Warning:</strong> The file size limit is 10MB.
</Callout>

<Callout type="warning">
  <strong>Warning:</strong> Authorization header must be provided
</Callout>

Optional query parameters:

- `to` - The folder to which the file should be uploaded in supabase. If not provided, the file will be uploaded to the `images` folder.

Response

```json
{
  "path": "assets/a-long-random-string-randomUUID-filename.extension"
}
```

To use the uploaded file, you can use the `path` returned in the and prefix it with `STORAGE_BUCKET_URL` environment variable.

## Uploading Multiple Files

Make a `multipart/form-data` request to the `/upload` route with the files to be uploaded in the `files` field.

<Callout type="warning">
  <strong>Warning:</strong> The file size limit is 10MB.
</Callout>

<Callout type="warning">
  <strong>Warning:</strong> Authorization header must be provided
</Callout>

Optional query parameters:

- `to` - The folder to which the file should be uploaded in supabase. If not provided, the file will be uploaded to the `images` folder.

Response

```json
{
  "paths": ["url1.jpg", "url2.jpg"]
}
```
