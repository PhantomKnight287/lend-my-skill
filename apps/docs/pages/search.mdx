# Search

These routes can be used to search services or jobs by using query, category or tags.

## Search using query

`GET /search`

Query parameters:

- `take`, to specify the numbers of results returned from the api
- `type`, either `Job` or `Service`
- `q`, the query

All the parameters except take is required.

Response

```json
{
  "services": [], //array of services, only if type is `Service`
  "next": 10, // only present if there are more results
  "jobs": [] //array of jobs, only if type is `Job`
}
```
